# 《山西古建保卫战》开发计划 · 40h 版

## 技术选型

**Phaser 3 + TypeScript + Vite**，单 HTML 页面，DOM/CSS 做 UI 叠加 Canvas。

不引入状态管理库、不搞复杂架构、不用对象池（40h 项目用不到）。

**美术方案：** 混合管线 — 核心角色/怪物/建筑使用 Aseprite 手绘 spritesheet（PNG-8 索引色，总 < 150 KB），背景/特效/粒子使用 Canvas2D 程序化生成（零体积）。风格参考：东方夜雀食堂（暖色调、精细像素、Q 版萌系）。详见 spec.md 第 15 章完整美术设计文档。

## 文件结构（极简）

```
src/
├── main.ts              # Phaser 启动配置
├── config.ts            # 所有数值常量（技能/怪物/波次/古建）
├── GameScene.ts         # 唯一场景，主循环 update()
├── Player.ts            # 玩家（移动 + 受击 + 拾取）
├── Monster.ts           # 怪物（一个 class，type 字段区分 5 种行为）
├── Building.ts          # 古建 4 血条
├── Skill.ts             # 技能（一个 class，id 字段区分 5 种逻辑）
├── SkillManager.ts      # 技能 CD + 自动释放
├── SpawnManager.ts      # 刷怪波次
├── ExpOrb.ts            # 经验球
├── HUD.ts               # 所有 UI（HP/血条/倒计时/经验/升级面板/弹窗/结算）
└── VFX.ts               # 简易特效（闪红/震动/浮动数字）
```

**原则：能不拆文件就不拆。** Monster 一个文件处理 5 种怪物，Skill 一个文件处理 5 种技能，HUD 一个文件处理全部 UI。

## 验收标准

完成以下全部即为 MVP 交付：

- [ ] WASD 移动流畅，镜头跟随
- [ ] 5 种怪物有不同外观和行为
- [ ] 古建 4 条血条独立显示、独立扣血
- [ ] 任意血条归零 → 游戏失败
- [ ] 5 个技能全自动释放，各 3 级，升满有明显差异
- [ ] 击杀 → 掉球 → 拾取 → 经验满 → 3 选 1 → 继续 闭环无 Bug
- [ ] 技能命中对应怪物回复对应血条（打怪修古建）
- [ ] 5 分钟完整关卡可玩，难度递增
- [ ] 胜利/失败结算面板
- [ ] 浏览器打开即玩，无需安装
