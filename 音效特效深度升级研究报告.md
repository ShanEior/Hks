## 2. 维度一：Travel 飞行层

### 2.1 现状：全部缺失

```
当前 8 技能反馈链：

施放(Cast) ──── [                 静默                  ] ──── 命中(Hit)
   ✅音效                   ❌ 无飞行音                      ✅音效
   ✅特效                                                    ✅特效
```

参考 Diablo 4（GDC 2024 官方演讲）：

```
施放(Cast) ── Whoosh ── 飞行(Travel) ── Impact ── 命中(Hit)
  一次性                    循环音频                   一次性
```

**这就是为什么很多技能"发出去很帅，打中很帅，中间像空气"**——玩家听到的是两个孤立事件，而非完整事件。

### 2.2 标杆参考

| 游戏 | Travel 实现 | 技术 |
|------|------------|------|
| **Diablo 4** | 火球循环 whoosh + 独立 travel 层 | 同源录音处理为长循环 |
| **Hades** | 血石投掷：短"嗖-砰"双重奏，100-300ms | 多普勒降频 + 立体声像移动 |
| **League of Legends** | 金克丝火箭/伊泽瑞尔 Q：独特飞行噪音 | 拟音+重度后期处理 |

### 2.3 四原型 Travel 声音系统

所有飞行音效可归为 4 种原型，Web Audio API 直接合成：

| 原型 | 感觉 | 技术 | 时长 |
|------|------|------|------|
| **嗖声 (Whoosh)** | 快速飞行/推力 | 带通滤波噪声频率下扫 2000→300Hz | 0.2-0.8s |
| **嗡声 (Buzz)** | 魔法能量球 | 锯齿波振荡器 + 噪声，恒定带通 | 0.5-3.0s |
| **嘶声 (Hiss)** | 毒气/酸性 | 高通白噪声 4000Hz+，快速衰减 | 0.1-0.4s |
| **噼啪声 (Crackle)** | 闪电/电弧 | 短脉冲方波 400-2000Hz + 极端高通噪声 | 0.05-0.2s |

### 2.4 8 技能 → 原型映射

| 技能 | 投射物行为 | Travel 原型 | 理由 |
|------|-----------|------------|------|
| `wood_reinforce` | 快速穿透光束 | **嗖声 A** | 木梁推力感，快直线 |
| `stone_repair` | 瞬发脉冲/AOE | **无** | 非投射物，脉冲瞬间生效 |
| `waterproof` | 多目标溅射弹 | **嘶声 C + 嗖声 A** | 液体投射物湿感+飞行感 |
| `insect_control` | 持续毒雾区域 | **嘶声 C** | 毒气持续雾气声 |
| `painting_restore` | 追踪弹跳弹丸 | **嗡声 B** | 魔法弹丸能量嗡声 |
| `repair_field` | 静态跟随光环 | **无** | 无投射物行为 |
| `whirlwind_slash` | 弧形风刃 | **嗖声 A** | 风刃锐利切割呼啸 |
| `chain_lightning` | 瞬间连锁 | **噼啪声 D** | 电弧爆裂，极快 |

**需要 Travel 的 6 个技能中有 5 个**（stone_repair 除外）。

### 2.5 Web Audio 实现（约 130 行）

核心：带通滤波噪声 + 频率扫频。

```javascript
// 嗖声原型 — ~20 行核心逻辑
function playTravelWhoosh(ctx, duration, startFreq, endFreq, volume) {
  const now = ctx.currentTime;
  const bufSize = Math.ceil(ctx.sampleRate * duration);
  const buffer = ctx.createBuffer(1, bufSize, ctx.sampleRate);
  const data = buffer.getChannelData(0);
  for (let i = 0; i < bufSize; i++) data[i] = Math.random() * 2 - 1;

  const noise = ctx.createBufferSource();
  noise.buffer = buffer;
  const bp = ctx.createBiquadFilter();
  bp.type = 'bandpass';
  bp.frequency.setValueAtTime(startFreq, now);
  bp.frequency.exponentialRampToValueAtTime(endFreq, now + duration);
  bp.Q.setValueAtTime(0.4, now);

  const gain = ctx.createGain();
  gain.gain.setValueAtTime(0, now);
  gain.gain.linearRampToValueAtTime(volume, now + 0.01);
  gain.gain.exponentialRampToValueAtTime(0.001, now + duration);

  noise.connect(bp).connect(gain).connect(masterOut);
  noise.start(now);
  noise.stop(now + duration + 0.05);
}
```

**性能**：5 个同时飞行投射物 × 3 节点 = 15 额外 AudioNode，远在 24 语音池预算内。`DynamicsCompressorNode` 主总线自动处理叠加保护。

## 3. 维度二：怪物受击反应

### 3.1 现状 vs 标杆

| 技术 | 本作当前 | Vlambeer 标准 | Dead Cells |
|------|---------|-------------|------------|
| 命中闪光 | **60-100ms** 白闪 | **1 帧(33ms)** | 1-2 帧 |
| 挤压/拉伸 | **0.75/1.3 (25%)** | — | 无（位置位移替代） |
| 击退 | ✅ 4 档 (60-350) | 3px/帧，子弹方向 | 按武器重量 |
| 命中停顿 | 全局减速 | 局部 10-20ms | Stagger（不冻结） |
| 死亡差异化 | 统一 3 阶段 | 33%爆炸+尸体物理 | 材质分解 |

### 3.2 三个关键发现

**发现 1：受击闪白太长。**

当前 Monster.ts `flashDuration = amount >= 30 ? 100 : 60`（ms）。
Vlambeer 用 1 帧（33ms@30fps）。问题是白光持续太久导致视觉疲劳——100ms 白闪后还有红色 Tint 延迟回调，总共可能 180ms 叠加。

**修复**：缩短到 `amount >= 30 ? 50 : 33`（1-2 帧@60fps）。

**发现 2：挤压/拉伸变形过大。**

当前 `scaleX * 0.75 / scaleY * 1.3` = 25% 形变。跨游戏研究共识：10-15% 是甜点区。25% 在 5+ 怪物同时受击时产生"抖动感"而非"冲击感"。

**修复**：改为 `0.88 / 1.12`（12% 形变），重击改为 `0.85 / 1.15`（15%）。

**发现 3：缺少轻重命中差异化。**

当前所有命中用相同变形参数，仅闪白颜色按伤害分档。但 Dead Cells 通过 Breach/Stagger 双系统创造了清晰的轻重武器反馈差异。

**修复**：轻击(<15 dmg)跳过分层变形，直接位移。重击(≥30)用全量变形 + 微击退增强。致命一击保留 5x 死亡膨胀。

### 3.3 改动清单

| 文件 | 内容 | 行数 |
|------|------|------|
| `Monster.ts` | 闪白缩短(100→50, 60→33) | 1 |
| `Monster.ts` | 挤压/拉伸减少(0.75→0.88, 1.3→1.12) | 1 |
| `Monster.ts` | 轻击跳过分层变形 | ~5 |
| **合计** | | **~7 行** |

## 4. 维度三：Hit Stop 架构

### 4.1 当前架构问题

```typescript
// CombatFeel.ts — 当前实现
update(delta): number {
  if (hitStopRemaining > 0) {
    hitStopRemaining -= delta;
    return 0.05;  // ← 全局减速到 5%，不是真正冻结
  }
  return delta;
}

// GameScene.ts — 当前用法
effectiveDelta = combatFeel.update(delta, time);
player.update(delta);              // ✅ 全速（正确）
allMonsters[i].update(effectiveDelta); // ❌ 全场 200 怪一起减速
```

**3 个问题**：

1. **全局减速而非单体冻结** —— 打中一只白蚁，全场 199 只怪同时减速 5%。像 Hollow Knight 和格斗游戏的 Hitlag 都是"仅被击中的角色冻结"。

2. **普攻触发 66ms 减速** —— `AUTO_ATTACK_CONFIG.damage = 22`，命中时 damage ≥ 15 触发 `medium` 档（66ms）。每 0.7s 自动发射的弓箭在持续刹车。

3. **冷却 150ms 让多段技能只停一次** —— 闪电链 4 跳在 90ms 间隔内全部被冷却吞掉。

### 4.2 对标研究

| 游戏 | 方式 | 时长 | 对称性 |
|------|------|------|--------|
| **Hades** | 全局时间缩放 | 轻 33ms / 重 100ms | 对称（Cast 不用） |
| **Hollow Knight** | 单怪冻结 | 普攻 50-80ms / 技艺 100-133ms | 不对称（玩家不冻） |
| **Dead Cells** | **完全不冻**，用 Stagger 延迟敌人攻击帧 | — | — |
| **Smash Bros** | 对称 Hitlag | 15 帧(250ms) | 双方冻结 |

### 4.3 推荐方案：单怪冻结

```typescript
// Monster.ts — 每个怪物独立 freezeTimer
freezeTimer: number = 0;

update(time, dt): void {
  if (this.freezeTimer > 0) {
    this.freezeTimer -= dt * 1000;
    this.drawHpBar();
    return; // 跳过 AI/移动/攻击
  }
  // 正常更新
}

takeDamage(amount): boolean {
  const tier = amount >= 60 ? 'heavy' : amount >= 30 ? 'medium' : 'light';
  this.freezeTimer = HIT_STOP_CONFIG[tier].freezeMs;
  // 击退仍然生效（冻结期间怪物仍被推开）
}
```

**普攻特例**：普攻命中传 `freezeMs = 0`，不冻结。

**CombatFeel 保留**：仅用于 VFX（冲击光环）+ 镜头冲击，不再管时间控制。

### 4.4 新的 Hit Stop 分级

| 等级 | 伤害阈值 | 冻结 ms | 帧(60fps) | 适用 |
|------|---------|---------|-----------|------|
| none | — | 0 | 0 | 普攻 |
| light | 1-14 | 33 | 2 | 区域DOT/溅射 |
| medium | 15-29 | 50 | 3 | 普通技能命中 |
| heavy | 30-59 | 83 | 5 | 暴击/闪电链 |
| ultra | 60+ | 150 | 9 | Boss出场/死亡 |

### 4.5 改动清单

| 文件 | 内容 | 行数 |
|------|------|------|
| `config.ts` | HIT_STOP_CONFIG 新增 `none` 档 | ~3 |
| `Monster.ts` | 新增 `freezeTimer` + 冻结跳过逻辑 | ~12 |
| `Monster.ts` | takeDamage 设置 freezeTimer | ~5 |
| `GameScene.ts` | 移除 effectiveDelta 逻辑，恢复 raw delta | ~3 |
| `CombatFeel.ts` | 简化为仅 VFX+镜头，不再返回 effectiveDelta | ~10 |
| **合计** | | **~33 行** |

## 5. 维度四：视觉优先级

### 5.1 问题诊断

当 50+ 怪物同屏且 4-6 个技能同时触发时：
- 每只怪受击 → burst(3) + 方向粒子(3) + floatText + shockwave → 视觉噪声叠加
- 技能自身的 Cast VFX（多层震波/水纹/彩色 burst）与怪物受击 VFX 叠加
- Soulstone Survivors 经历的"玩家看不清自己是否受伤"困境可能重现

### 5.2 行业方案

| 方案 | 来源 | 做法 |
|------|------|------|
| **敌人留白** | Vampire Survivors | 怪物零 per-hit VFX，只靠 knockback+数字。玩家技能 VFX 才是主角 |
| **优先级层级** | Diablo 4 | 敌人预警 > 玩家技能 > 环境效果。低优先级降低 opacity |
| **伤害阈值** | Death Must Die | 低于某阈值的伤害不生成 per-hit 粒子 |

### 5.3 推荐方案：吸血鬼幸存者哲学 + 伤害阈值

```
怪物受击 VFX 分级：

伤害 < 8： 仅数字（不生成粒子）
伤害 8-19： burst(2) + 数字
伤害 20-39：burst(3) + 方向粒子(2) + 数字
伤害 40+：  完整 burst(3) + 方向粒子(3) + shockwave + 数字
```

**最简单的实现**：在 `VFX.hitMonster` 中加一个 `if (damage < 8) return` 的早期出口（仅数字仍然生成）。一行改动。

### 5.4 改动清单

| 文件 | 内容 | 行数 |
|------|------|------|
| `VFX.ts` | hitMonster 添加 damage 阈值 | ~3 |
| `config.ts` | 新增 `VFX_THRESHOLD` 常量 | ~3 |
| **合计** | | **~6 行** |

## 6. 维度五：升级反馈

### 6.1 现状

升级时：
- VFX.levelUp(): 双层金光 + 20粒子 + 双shockwave + "Level X"文字 ✅
- SoundManager.levelUp(): 三音阶 ✅
- 但**战斗不停** —— 怪物继续进攻，玩家注意力被抢走

### 6.2 参考

| 游戏 | 做法 |
|------|------|
| Hades | 世界暂停，神出现 |
| Death Must Die | 时间冻结 |
| Vampire Survivors | 直接暂停战斗，升级界面全屏 |

### 6.3 方案

非技能选择面板的普通升级（`showLevelNotify`）：战斗不停是设计意图（割草游戏不应频繁暂停）。但可在 `checkLevelUp` 中调用 CombatFeel 触发 50ms ultra Hit Stop + 镜头微缩放，切开"战斗"和"奖励"两个瞬间。

**无需额外代码**——前一步 Hit Stop 架构修复后自然会生效。
