## 2. 全量审计：8技能 × 4维度覆盖矩阵

32 个槽位逐个核查，✅ = 就位，❌ = 缺口。

### 2.1 老 5 技能

| 维度 | 木构加固 | 石材修补 | 防水封护 | 防虫处理 | 彩绘修复 |
|------|:---:|:---:|:---:|:---:|:---:|
| **Cast VFX** | ✅ skillWood | ✅ skillStone | ✅ skillWater | ✅ skillInsect | ✅ skillPaint |
| **Cast Sound** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Hit VFX** | ✅ woodImpact | ❌ 通用灰burst | ✅ waterImpact | ⚠️ insectTick有/spore❌ | ✅ paintImpact |
| **Hit Sound** | ❌ | ❌ | ❌ | ❌ | ❌ |

**老 5 技能缺口**：1 个 Hit VFX（stonePulse）+ 5 个 Hit Sound

### 2.2 新 3 技能（pipi3）

| 维度 | 修复法阵 | 旋风斩 | 雷电链 |
|------|:---:|:---:|:---:|
| **Cast VFX** | ✅ skillRepairField | ✅ skillWhirlwind | ✅ skillLightningCast |
| **Cast Sound** | ❌ 错配water | ❌ 错配stone | ❌ 错配water |
| **Hit VFX** | ✅ repairFieldPulse | ✅ whirlwindHit | ❌ 仅电弧线/无impact爆 |
| **Hit Sound** | N/A（纯治疗） | ❌ | ❌ |

**新 3 技能缺口**：3 个 Cast Sound + 2 个 Hit Sound + 1 个 Hit VFX（lightningImpact）

### 2.3 合计

```
32 槽位
├── ✅ 就位：21 个
└── ❌ 缺口：11 个
    ├── Cast Sound：3 个（repair / whirlwind / lightning）
    ├── Hit Sound： 7 个（wood / stone / water / insect / paint / whirlwind / lightning）
    └── Hit VFX：  1 个（lightningImpact）+ 2 个待优化（stonePulse / insectSpore）
```

### 2.4 玩家实际听到的

所有 7 个有伤害命中的技能——不管你是放出木梁、石材脉冲、水幕、药雾、颜料弹、旋风斩、还是雷电链——命中怪物时听到的都是 **Monster 内部同一声 `hitMonster()` 通用"咚"**。唯一区别是 Monster.takeDamage 自带去重（50ms），多段命中可能被吞掉部分。

这就是你说的"技能本身没有对应的音效和特效"——**Hit Sound 全军覆没**。

## 3. 标杆游戏对比

### 3.1 各游戏技能音效体系

| 游戏 | Cast | Travel | Impact | Kill | 设计哲学 |
|------|:---:|:---:|:---:|:---:|------|
| **Diablo 4** | ✅ | ✅ | ✅ | ✅ | GDC 2024 官方：3层音频+Living Audio随机变异 |
| **Hades** | ✅ | ✅ | ✅ | ✅ | 每层独立合成层+神祇boon叠加 |
| **Death Must Die** | ✅ | - | ✅ | ✅ | 9神体系每神独立音色签名 |
| **Dead Cells** | ✅ | - | ✅ | ✅ | 材质音+武器音+敌人叫声 3层叠加 |
| **本作(当前)** | 5/8✅ | ❌ | **0/7** | ⚠️ 通用Monster | 仅 Cast + 通用 hitMonster |

**差距是数量级的**。不是"差一点"，是"缺少一整层"。

### 3.2 Hades Cast 释放精确参数（60fps 逐帧）

| 帧 | 时间 | 事件 |
|----|------|------|
| 0 | 0ms | 输入注册，角色进入 cast_start |
| 1-4 | ~0-67ms | **蓄力**：scaleX~1.08/scaleY~0.94 |
| 5-8 | ~67-133ms | **释放**：投射物生成，角色 30-40% 白色闪 2-3 帧 |
| 9-12 | ~133-200ms | **恢复**，可取消 |

**关键纠正**：Hades Cast **不使用 Hit Stop**（仅武器命中帧用）。角色缩放是预备动作（1.08→1.0），不是释放后的 pop。

### 3.3 Diablo 4 技能音频三层体系（GDC 2024 官方）

Firebolt 例：

| 层 | 类型 | 实现 |
|----|------|------|
| CAST | 一次性 | 燃烧法杖实地录制，编辑处理呼啸声 |
| TRAVEL | 循环 | 投射物飞行持续火焰循环 |
| IMPACT | 一次性 | 命中编辑处理爆炸声 |

**Living Audio 三原则**：
1. **随机变异**：每层有多个变体，音高 ±5-10%、音量 ±3-5dB，永远不重复
2. **独立频段**：火=中低频、冰=高频水晶、雷=尖锐瞬态
3. **频段 Ducking**：同时发声时不是降音量，是接管特定频段空间

**本作可对标**：当前 `SKILL_AUDIO` 表已有 8 技能的频段+波形参数——**缺的是 cast/hit 两层参数化和对应的播放方法**。

### 4.3 现有调用修正

**SkillManager.ts — `castSkill()` 音效映射修正**：

```
当前                        → 修正
skillWater (repair)         → skillRepairField()
skillStone (whirlwind)      → skillWhirlwind()
skillWater (lightning)      → skillLightning()
```

**SkillManager.ts — `handleProjectileHit()` 命中音补入**：

```
当前                        → 补入
VFX.woodImpact()            + SoundManager.skillWoodHit()
VFX.paintImpact()           + SoundManager.skillPaintHit()
VFX.whirlwindHit()          + SoundManager.skillWhirlwindHit()
```

**SkillManager.ts — `castChainLightning()` 弹射命中补入**：

```
VFX.lightningArc()          + SoundManager.skillLightningHit()（每次弹射）
                            + VFX.lightningImpact()（命中爆发）
```

**SkillManager.ts — `castWaterproof()` 命中补入**：

```
target.takeDamage()         + SoundManager.skillWaterHit()
```

**SkillManager.ts — `damageInRadius()` 石材脉冲命中补入**：

```
takeDamage()                + SoundManager.skillStoneHit()
                            + VFX.stonePulse()（替代当前灰burst）
```

**SkillManager.ts — `updateZones()` 药雾/孢子命中补入**：

```
VFX.insectTick()            + SoundManager.skillInsectHit()
孢子 takeDamage()            + VFX.insectSpore()（替代当前通用burst）
```

### 4.4 设计问题修复

**升级闪光**（`VFX.ts levelUp()`）：

```
删除：VFX.flash(scene, 80)
替换：双重金色冲击波（内圈40 + 外圈100，错时60ms）
保留：20 粒子螺旋上升 + 音阶音效
新增：Hit Stop 60ms 在 GameScene.ts 升级检测处
```

**Cast Burst 统一释放入口**（`SkillManager.ts castSkill()` 头部新增 5 行）：

```typescript
// 所有技能共享的释放瞬间反馈
player.sprite.setScale(1.06, 1.06);
this.scene.tweens.add({ targets: player.sprite, scaleX: 1, scaleY: 1, duration: 50, ease: 'Back.easeOut' });
VFX.shake(this.scene, 0.003, 40);
const flash = this.scene.add.circle(player.x, player.y, 4, 0xffffff, 0.7);
this.scene.tweens.add({ targets: flash, radius: 14, alpha: 0, duration: 100,
  onComplete: () => flash.destroy(), onStop: () => { if (flash.active) flash.destroy(); } });
```

### 4.5 技能反馈映射表（config.ts）

**本阶段建立，为未来扩展打基础**：

```typescript
export const SKILL_FEEDBACK_MAP: Record<SkillId, {
  castSound: string;   // SoundManager 方法名
  castVFX: string;     // VFX 方法名
  hitSound?: string;   // 技能专属命中音方法名（可选）
  hitVFX?: string;     // 技能专属命中 VFX 方法名（可选）
}> = {
  wood_reinforce:   { castSound: 'skillWood',    castVFX: 'skillWood',
                      hitSound: 'skillWoodHit',  hitVFX: 'woodImpact' },
  stone_repair:     { castSound: 'skillStone',   castVFX: 'skillStone',
                      hitSound: 'skillStoneHit', hitVFX: 'stonePulse' },
  waterproof:       { castSound: 'skillWater',   castVFX: 'skillWater',
                      hitSound: 'skillWaterHit', hitVFX: 'waterImpact' },
  insect_control:   { castSound: 'skillInsect',  castVFX: 'skillInsect',
                      hitSound: 'skillInsectHit',hitVFX: 'insectSpore' },
  painting_restore: { castSound: 'skillPaint',   castVFX: 'skillPaint',
                      hitSound: 'skillPaintHit', hitVFX: 'paintImpact' },
  repair_field:     { castSound: 'skillRepairField', castVFX: 'skillRepairField' },
  whirlwind_slash:  { castSound: 'skillWhirlwind',   castVFX: 'skillWhirlwind',
                      hitSound: 'skillWhirlwindHit', hitVFX: 'whirlwindHit' },
  chain_lightning:  { castSound: 'skillLightning',    castVFX: 'skillLightningCast',
                      hitSound: 'skillLightningHit',  hitVFX: 'lightningImpact' },
};
```

**这不是为了本阶段直接查表调度，而是**：
1. 作为 `castSkill()` 和 `handleProjectileHit()` 的**文档化映射** — 新加技能时先看这张表
2. 后续重构时 `castSkill()` 的 switch/case 可以直接简化为此表查表调用
3. 现在就能用 — `castSkill()` 里的 VFX/Sound 调用直接从表中取值而非手写字符串
