# 《山西古建保卫战》— 项目核心文档

## 项目身份

- **名称**：山西古建保卫战 (Shanxi Ancient Architecture Defense)
- **类型**：像素风 Roguelike 割草生存小游戏
- **技术栈**：Phaser 3 + TypeScript + Vite
- **分支**：`dev-aaa`（个人开发分支，合入 `master` 需协作方审批）
- **上游仓库**：https://github.com/ShanEior/Hks

---

## 核心设计理念（不可违背）

### 1. 极简架构 — "能不拆文件就不拆"

- 一个 class 处理多种变体（Monster 一个文件处理 5 种怪物，Skill 一个文件处理 5 种技能）
- 不引入状态管理库、不搞复杂架构、不用对象池
- 新增功能优先扩展现有文件，拆分只在文件超过 ~800 行且职责明显不同时才考虑
- **判断标准**：如果新功能可以放进现有 class 的一个 `switch/case` 或 `if/type` 分支里，就不要新建文件

### 2. 零外部资源 — 全代码生成

- 所有美术：`ArtGen.ts` 用 Canvas2D 程序化生成像素纹理，零外部图片文件
- 所有音效：`SoundManager.ts` 用 Web Audio API 合成，零外部音频文件
- **包体积预算**：JS 包 < 2MB（当前约 1.5MB）
- 新增视觉/音效元素必须走代码生成路线，禁止引入 PNG/MP3 等外部资源

### 3. 像素风国风 — 视觉一致性

- 调色板统一使用 `config.ts` 中的 `PALETTE` 常量（金色/木色/暗木/朱砂/玉绿/石灰色/羊皮纸色）
- 精灵尺寸 16×16 ~ 48×48 为主，配合粗像素感
- UI 风格：木质纹理 + 金色边框 + 卷轴/牌匾造型
- 字体：系统 CJK 字体栈（`config.ts` 中的 `FONT` 预设）
- 新增纹理必须在 `ArtGen.ts` 中生成，使用统一的 `PX` 像素单位

### 4. 打怪修古建 — 核心战斗闭环

- 技能命中怪物 → 回复对应古建结构血条
- 这是本游戏最核心的差异化机制：攻击 = 防御
- 每个技能 Lv.2/Lv.3 增加回复能力，形成"越打越修"的正反馈
- 新增技能必须遵守此设计：有伤害就要考虑是否触发回血

### 5. 古建文化科普 — 教育属性

- 怪物类型映射真实山西古建威胁（白蚁=蛀蚀，风蚀=风沙，酸雨=化学风化，火=火灾，冻融=冰胀）
- 首次遭遇弹出科普弹窗（`HUD.showMonsterPopup`）：灾害背景 + 游戏影响
- 新增怪物必须附带科普内容
- 保持"游戏 + 文化传播"的双重属性

### 6. 自动战斗 + 策略选择

- 玩家普攻自动发射、技能自动 CD 释放
- 玩家核心决策：走位躲避 + 升级时从 3 个选项中选 1 个
- 不要让玩家手动释放技能或手动瞄准
- 新增战斗机制应保持"自动执行 + 玩家策略选择"的模式

### 7. 数值透明 — 单一真相源

- 所有游戏常量在 `config.ts` 中统一定义
- 不可以在其他文件中硬编码数值
- 新增怪物/技能/波次数据必须在 `config.ts` 中添加配置项
- 调参只改 `config.ts`

### 8. 单场景架构

- 全游戏只有一个主场景 `GameScene`（外加 `MenuScene` 和 `TutorialScene` 两个辅助场景）
- 新增关卡/模式优先在 `GameScene` 中通过配置切换实现，而非新建 Scene

## 禁止事项

- ❌ 不要引入新的 npm 依赖（保持依赖只有 phaser + typescript + vite）
- ❌ 不要添加 PNG/MP3/WAV 等外部资源文件
- ❌ 不要在 `config.ts` 以外的文件硬编码游戏数值
- ❌ 不要新建超过 3 个文件的 feature（思考是否可以在现有文件中扩展）
- ❌ 不要修改 `main.ts` 的启动配置（除非有充分理由）
- ❌ 不要改变"自动攻击 + 自动技能"的战斗模式
- ❌ 不要删除或弱化科普弹窗系统
- ❌ 不要让包体积超过 2MB

## 子智能体

本项目定义了 3 个专项子智能体，用于辅助开发：

### game-reviewer
Phaser 3 游戏代码专项审查。检查 update() 中对象创建（new 在循环里？）、Tween 泄漏、Phaser 生命周期、事件监听未解绑、碰撞检测性能。输入 diff 或指定文件，输出严重/警告/建议三级。

### balance-tuner
游戏数值平衡分析。读取 config.ts 中的全部数值，计算怪物 DPS vs 古建血量、技能伤害 vs 怪物血量、经验曲线 vs 升级频率、6 阶段压力梯度，输出异常点和调参建议。

### pixel-artist
像素艺术视觉一致性检查。审查 ArtGen.ts 纹理是否符合像素风格、调色板是否统一、动画帧是否流畅、UI 元素对齐裁剪问题。输入截图或 ArtGen.ts，输出改进建议。
